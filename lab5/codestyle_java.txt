Кодстайл компании «Пупсики» для Java

• Должна присутствовать обработка исключений
• Явная статическая типизация обязательна
• Захардкоженные данные убираем, заменяем вводом с клавиатуры
• Максимально оптимизируем и минимизируем код
• Отступы только табуляцией
• Предпочтительный размер табуляции — 4 пробела
• Максимальная длина строки — 64 символа, так как пишем код на небольших ноутбуках
• Перенос строки перед бинарным оператором для читаемости
• Пустые строки — между классами 2 строчки, между логическими разделами 1 строка
• Кодировка исходного файла — UTF-8
• Импорты — отдельно в каждой строке. Импорты помещаются в начале файла, сразу после package и перед объявлениями классов и констант
• Кавычки — двойные для String, одинарные только для char
• Пробелы. Избегаем пробелы внутри конструкций. Ставим пробелы после знаков препинания. Между бинарными операторами пробелы с каждой стороны
• Не используем составные инструкции
• Тело цикла только с новой строки
• Все комментарии пишем на русском языке, с маленькой буквы, в отдельной строке перед комментируемой строчкой
• Все классы, интерфейсы, enum и методы должны иметь Javadoc с параметрами (обязательно)
• Константы объявляются в верхнем регистре
• Именование:
    Переменные и методы — lowerCamelCase
    Классы, интерфейсы, enum — PascalCase
    Константы — UPPER_CASE
    Приватные поля — lowerCamelCase
• Import * запрещен
• Предпочитаем final для неизменяемых переменных и параметров
• При написании программы руководствоваться принципами ООП
